Create Database (Examples)


	 CREATE DATABASE  database_name;

Example 1:  Create a Basic Database 

			This is the simplest way to create a database:

	CREATE DATABASE MyDatabase;
 
		This will create a new database named `MyDatabase` with default settings.

 Example 2:  Create a Database with Specific File Locations 

		You can specify where the database files (data and log files) should be stored. For example:

	 CREATE DATABASE  MyDatabase

	 ON  

	(
	     NAME  = MyDatabase_Data,
	     FILENAME  = 'C:\SQLData\MyDatabase_Data.mdf',

	     SIZE  = 5MB,
	     MAXSIZE  = 100MB,

	    FILEGROWTH = 5MB
	)

	 LOG ON  

	(
	     NAME  = MyDatabase_Log,
	     FILENAME  = 'C:\SQLData\MyDatabase_Log.ldf',

	     SIZE  = 2MB,
	     MAXSIZE  = 50MB,

	     FILEGROWTH  = 2MB
	);
 

	- `NAME`: Specifies the logical name of the data or log file.

	- `FILENAME`: Specifies the physical location of the data or log file.

	- `SIZE`: Sets the initial size of the file.

	- `MAXSIZE`: Specifies the maximum size the file can grow to.

	- `FILEGROWTH`: Determines how much the file should grow when more space is needed.

 Example 3:  Create a Database with Collation and Options 

	 CREATE DATABASE  MyDatabase

	 COLLATE  Latin1_General_CI_AS

	 ON  

	(
	     NAME  = MyDatabase_Data,
	     FILENAME  = 'C:\SQLData\MyDatabase_Data.mdf',

	     SIZE   = 10MB,
	     MAXSIZE  = 200MB,

	     FILEGROWTH  = 10MB
	)

	 LOG ON 

	(
	     NAME  = MyDatabase_Log,
	     FILENAME  = 'C:\SQLData\MyDatabase_Log.ldf',

	     SIZE  = 5MB,
	     MAXSIZE  = 100MB,

	     FILEGROWTH  = 5MB
	)

	 WITH  

	     RECOVERY  = FULL,  -- Recovery model
		
	     COMPATIBILITY_LEVEL  = 110; -- Compatibility with SQL Server 2012
 
 Example 4:  Create a Database with Filegroups 

	 CREATE DATABASE  MyDatabase

	 ON 

	PRIMARY ( 

	     NAME  = MyDatabase_Data,
	     FILENAME  = 'C:\SQLData\MyDatabase_Data.mdf',

	     SIZE  = 10MB,
	     MAXSIZE  = 100MB,

	     FILEGROWTH  = 10MB
	),

	 FILEGROUP FG2 ( 

	     NAME  = MyDatabase_FG2,
	     FILENAME =  'D:\SQLData\MyDatabase_FG2.ndf',

	     SIZE  = 10MB,
	     MAXSIZE  = 100MB,

	     FILEGROWTH  = 10MB
	)

	 LOG ON ( 

	     NAME  = MyDatabase_Log,
	     FILENAME  = 'E:\SQLData\MyDatabase_Log.ldf',

	     SIZE  = 5MB,
	     MAXSIZE  = 50MB,

	     FILEGROWTH  = 5MB
	);

		-  PRIMARY  is the main filegroup where the primary data file resides.

		-  FG2  is a secondary filegroup where additional data files can be placed.

Example 5:  Create a Database with Specific Auto Close and Auto Shrink Settings 

	 CREATE DATABASE  MyDatabase

	ON

	(
	     NAME  = MyDatabase_Data,
	     FILENAME  = 'C:\SQLData\MyDatabase_Data.mdf',

	    SIZE = 10MB,
	    FILEGROWTH = 10MB
	)

	LOG ON

	(
	    NAME = MyDatabase_Log,
	    FILENAME = 'C:\SQLData\MyDatabase_Log.ldf',

	    SIZE = 5MB,
	    FILEGROWTH = 5MB
	)

	WITH

	- AUTO_CLOSE = ON,  -- Automatically close the database when no users are connected
	- AUTO_SHRINK = OFF;  -- Disable auto shrink
	- AUTO_CLOSE : Automatically closes the database when there are no active connections, which can save system resources.
	- AUTO_SHRINK : Automatically reduces the size of the database when space is freed.

Example 6:  Create a Database with a Specific Recovery Model 

	CREATE DATABASE MyDatabase

	ON

	(
	    NAME = MyDatabase_Data,
	    FILENAME = 'C:\SQLData\MyDatabase_Data.mdf',

	    SIZE = 10MB,
	    FILEGROWTH = 10MB
	)

	LOG ON

	(
	    NAME = MyDatabase_Log,
	    FILENAME = 'C:\SQLData\MyDatabase_Log.ldf',

	    SIZE = 5MB,
	    FILEGROWTH = 5MB
	)

	WITH RECOVERY = FULL;
 
Example 7:  Create a Database Using a Template Database 

	You can also create a new database based on the structure of an existing template database (such as `model`).

		CREATE DATABASE MyDatabase

	ON (NAME = MyDatabase_Data, FILENAME = 'C:\SQLData\MyDatabase_Data.mdf')

		LOG ON (NAME = MyDatabase_Log, FILENAME = 'C:\SQLData\MyDatabase_Log.ldf')

	AS COPY OF model;
