Alter Table (Examples)


  1.  Alter Table to Add a Column 

	To add a new column to an existing table:

	ALTER TABLE Employees

	 ADD  DateOfBirth DATE;
 
  2.  Alter Table to Modify a Column 

	To modify the data type of a column:

 
	ALTER TABLE Employees

	 ALTER COLUMN  EmployeeName VARCHAR(200);
 
 3.  Alter Table to Drop a Column 

	To remove a column from an existing table:
 
	ALTER TABLE Employees

	 DROP COLUMN  DateOfBirth;
 
  4.  Alter Table to Rename a Column (SQL Server 2016+) 

	To rename an existing column in SQL Server 2016 or later:
 
	 EXEC sp_rename  'Employees.EmployeeName', 'FullName', 'COLUMN';

	Note that `sp_rename` is used for renaming, not the `ALTER` statement.

  5.  Alter Table to Add a Constraint 

	To add a primary key constraint to a column:
 
	ALTER TABLE Employees

	 ADD CONSTRAINT  PK_EmployeeID  PRIMARY KEY  (EmployeeID);

	To add a foreign key constraint:

 	ALTER TABLE Orders

	 ADD CONSTRAINT  FK_Orders_EmployeeID  FOREIGN KEY  (EmployeeID)

	REFERENCES Employees(EmployeeID);
 
  6.  Alter Table to Drop a Constraint 

	To drop a constraint, such as a foreign key constraint:

	ALTER TABLE Orders

	 DROP CONSTRAINT  FK_Orders_EmployeeID;

  7.  Alter Table to Rename a Table 

	To rename an existing table:

	 EXEC sp_rename  'Employees', 'Staff';
 
  8.  Alter Table to Change Column's NULLability 

	To change a column from `NULL` to `NOT NULL`:

 	ALTER TABLE Employees

	 ALTER COLUMN  EmployeeName VARCHAR(100) NOT NULL;

	To allow a column to accept `NULL` values:

	ALTER TABLE Employees

	 ALTER COLUMN  EmployeeName VARCHAR(100) NULL;

  9.  Alter Table to Add a Default Value 

	To add a default constraint to a column:

	ALTER TABLE Orders

	 ADD CONSTRAINT  DF_OrderDate  DEFAULT  GETDATE() FOR OrderDate;
 
  10.  Alter Table to Drop a Default Constraint 

	To drop a default constraint:
 
	ALTER TABLE Orders

	 DROP CONSTRAINT  DF_OrderDate;
