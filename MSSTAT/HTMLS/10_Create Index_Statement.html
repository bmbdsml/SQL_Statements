Create Index (Examples)

	CREATE [UNIQUE] INDEX index_name

	ON table_name (column_name1, column_name2, ...);

 Example 1:  Create a Non-Clustered Index 

		A non-clustered index stores the index in a separate structure from the actual data, 
		
		which helps speed up queries that don't need to read all the rows of the table.

 
		CREATE INDEX idx_ProductName ON Products (ProductName);
 
		- This creates a non-clustered index named `idx_ProductName` on the `ProductName` column in the `Products` table.

Example 2:  Create a Clustered Index 

		A  clustered index determines the physical order of the data in the table, meaning the rows are stored in the order 
		
		of the indexed column(s). 

		A table can only have one clustered index because the data can only be sorted one way.

 
	CREATE CLUSTERED INDEX idx_ProductID ON Products (ProductID);
 
	- This creates a clustered index on the `ProductID` column in the `Products` table.

	- Since `ProductID` is likely the primary key, this will be the default clustered index unless otherwise specified.


 Example 3:  Create a Unique Index 

			A unique index ensures that all values in the indexed column(s) are unique.
 
	CREATE UNIQUE INDEX idx_Products_SerialNumber ON Products (SerialNumber);
	 
		- This creates a unique index on the `SerialNumber` column in the `Products` table, 
		
		  ensuring no two products have the same serial number.

 Example 4:  Create a Composite (Multi-Column) Index 

		A composite index is an index on multiple columns. 

		It can improve query performance for queries that filter or sort by these columns.
 
	CREATE INDEX idx_ProductName_Manufacturer ON Products (ProductName, Manufacturer);
 
	- This creates a non-clustered index on both the `ProductName` and `Manufacturer` columns in the `Products` table.
	
	- The order of the columns in the index matters, as it can affect query performance.

 Example 5:  Create a Filtered Index 

	A filtered index is an index that only includes rows that meet a certain condition. 

	This is useful when only a subset of the data is frequently queried.
 
	CREATE INDEX idx_Products_Available ON Products (ProductName) WHERE StockQuantity > 0;

	- This creates an index on the `ProductName` column for rows where the `StockQuantity` is greater than zero.
	
	- This can improve performance for queries that only retrieve products that are in stock.

 Example 6:  Create a Full-Text Index 

	A full-text index is used for full-text searching of textual data. 

	It allows for fast searching of large amounts of text data, such as looking for specific words or phrases.
 
	CREATE FULLTEXT INDEX ON Products (ProductDescription) KEY INDEX idx_ProductID; 

		- This creates a full-text index on the `ProductDescription` column, using the `ProductID` 
		
		  index as the key index for the full-text search.
 

 Example 7:  Create a Spatial Index 

		A spatial index is used for columns that store spatial data (e.g., geometry or geography data types).
 
	CREATE SPATIAL INDEX idx_SpatialData ON Locations (LocationData);
 
		- This creates a spatial index on the `LocationData` column in the `Locations` table, which is useful 
		
		  for improving queries that involve geographic or geometric data.

 Example 8:  Create an Index on a Computed Column 

	You can create an index on a computed column, which is a column defined by a calculation from other columns.

	CREATE INDEX idx_ComputedPrice ON Products (Price * Quantity);
 
	- This creates an index on a computed column that calculates the price times quantity for the `Products` table.

 Example 9:  Create an Index with INCLUDE Clause 

	The `INCLUDE` clause allows you to include additional columns in the index, which are not part of the key, 
	
	but can help optimize certain queries that require those columns.

 	CREATE NONCLUSTERED INDEX idx_Products_Name_Price ON Products (ProductName) INCLUDE (Price, StockQuantity);
 
		- This creates a non-clustered index on the `ProductName` column, but also includes `Price` 
		
		  and `StockQuantity` in the index.
		  
		- The `INCLUDE` columns are not part of the key, but can help avoid the need for SQL Server to access the table data 
		
	 	  for queries that need those columns.

 Example 10:  Create a Filtered Index with Include Clause 

		A filtered index  with an `INCLUDE` clause can be even more efficient by creating an index that is 
		
		focused on a subset of data and includes additional non-key columns for performance.
 
	CREATE NONCLUSTERED INDEX idx_ActiveProducts ON Products (ProductName) WHERE IsActive = 1 INCLUDE (Price, StockQuantity);
	

		- This creates a non-clustered filtered index on the `ProductName` column, but only includes rows where `IsActive = 1`.
		
		- It also includes `Price` and `StockQuantity` for queries that need those columns, but without requiring a table lookup.

 Example 11:  Drop an Index 

			If you want to remove an index, you can use the `DROP INDEX` statement:

 		DROP INDEX idx_ProductName ON Products;
 
			- This removes the `idx_ProductName` index from the `Products` table.

  < Non-clustered index > 

	 Speeds up read operations by storing a separate index structure

  < Clustered index > 

	 Organizes the table data itself in a sorted order

  < Unique index > 

	 Ensures no duplicate values exist in the indexed columns

 < Composite index >

	 Indexes multiple columns together

 < Filtered index > 

	An index on a subset of data based on a condition

 < Full-text index >

	 Enables full-text search capabilities

 < Spatial index >

	 Used for spatial data types (e.g., geographical data)

  < Include clause >

	 Adds non-key columns to an index to improve query performance
